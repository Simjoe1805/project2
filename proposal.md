# What I’m building (one sentence):

A service-on-demand web app that connects users with nearby professionals (plumbers, HVAC technicians, electricians, locksmiths, appliance repair, and handymen), similar to Uber but for home services.

# Which API I’m using (name and URL):
Geoapify Places API – https://www.geoapify.com/places-api
(Optional backup) Google Maps Places API – https://developers.google.com/maps/documentation/places/web-service


# Why I chose this:
I’m interested in building something real-world and scalable (this could actually become a startup idea).
Service marketplaces like Uber and TaskRabbit are widely used, but there’s still fragmentation in home services.
The API allows me to fetch real location-based data, making the project dynamic instead of static.
It helps me practice front-end + API integration + user experience design.


# Core features (3–5 things it will do):
Service selection: Users choose a category (plumbing, HVAC, locksmith, etc.)
Location-based search: Fetch nearby providers using API (based on user input or location)
Service cards UI: Display providers with name, rating, distance, and type
Request button: Simulate booking/requesting a service (like “Call” or “Request Now”)
Simple filtering: Filter by service type or distance


# What I don’t know yet (learning goals):

How to parse and display JSON data dynamically in the DOM
Handling user location input vs automatic location
Managing API keys and errors (like failed requests or empty results)
